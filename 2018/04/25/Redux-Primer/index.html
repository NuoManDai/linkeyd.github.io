React-Redux入门

发表于 2018-04-25   |   分类于 react ， redux   |     |   阅读次数

Redux自述

当我们在开发React component的时候我们会遇到一个非常棘手的问题，由于React是一种单向数据流的特性，通过props自上而下传递数据在通过state触发组件渲染。
当我们的React项目越来越庞大涉及到组件之间的数据传递越来越复杂这时候我们就必须面对一个问题:怎么样才能够让数据在组件之间优雅的传递又能满足单向数据流的规则。
这时候我们就可以使用redux，它是一个容器相当于在react的顶层建立了一个Provider的容器，自上而下的对容器进行分发。从而满足了单向数据流的规则，将应用的数据进行统一管理，并且使组件之间的数据传递变得更加的便捷。

React-Redux模块安装

在React项目中使用redux我们将安装接下来这几个模块

Redux功能使用

Action

Action是redux中用来申明一个数据model，它是一个纯函数将描述一个保存到redux上的数据是一个什么样子的.
它的返回值是一个对象，必须默认的约定由type键和其他数据组成。而type键中的值将会为借来下reduce时创建一个新的redux state 起到至关重要的作用.

Reducers

它通过redux中combineReducers方法，类似触发器用于匹配由Action带来的type通过type进行删选让type值通过switch进行删选，再把对应的数据添加到redux state中构成新的树.
react-router-redux的包将会在reducers中使用，将路由添加进入新的redux state树中.
Reducers中常使用的函数有[...state,action.text]或者 Object.assgin(state,action.text)两种函数都是对原有的Object在不做任何修改的情况下对Object进行合并.

Store

Store是redux中的数据仓库,用于存放reducers产生的数据和设置Redux的中间件.

Provider

Provider是React-redux中的一个组件，用于将redux中store的数据保存在应用程序的最顶层的容器之中.

connect

connect是react-redux中的一个组件，用于连接component与redux,触发redux状态更新，并且从Provider中获取数据。它是一个带有实例方法的高阶组件。
